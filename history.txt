★★★ [버그 수정 - ROI 검사 시 MainView 멈춤] ★★★
"ROI 설정 후 검사 켜면 메인 화면만 멈추는 심각한 버그"
-------------------------------------------
상황: 검사 ROI를 설정한 뒤 Start Inspection을 누르면, 디버그 뷰는 갱신되는데 메인 이미지 뷰(MainView)만 멈춰서 더 이상 안 바뀜.

[원인 1 - 스레드 상태와 플래그 불일치(데드락)]
무슨 일이었나: 검사는 백그라운드 스레드(QThread)에서 돌아감. "지금 검사 중인가?"를 나타내는 _worker_busy와, 스레드가 "완전히 끝났는가"를 나타내는 isRunning()을 둘 다 보고 있었는데, 타이밍이 어긋나서 "검사는 끝났다고 생각하는데 아직 스레드 정리 중"인 순간에 다음 검사를 시작하려다 run_detection()에서 "아직 실행 중"이라 스킵 → 검사가 안 시작되는데 _worker_busy는 True로 남아 버려 영원히 멈춤.
해결:
- run_detection() 안의 isRunning() 체크 제거. "검사 끝남"은 QThread의 finished 시그널이 날아올 때만 _worker_busy = False로 하도록 _on_worker_finished()에서만 처리. _on_detection_result()에서는 _worker_busy를 건드리지 않음.
- Worker의 run() 안에 try-except를 넣어서, 예외가 나도 빈 결과라도 시그널로 보내서 메인 쪽이 멈추지 않게 함.

[원인 2 - contour 좌표 자료형 (가장 핵심)]
무슨 일이었나: ROI로 잘라서 검사한 뒤, 검출된 contour(윤곽) 좌표를 "전체 이미지 기준"으로 옮기려고 오프셋(rx, ry)을 더했음. 그런데 (정수 배열 + 실수 오프셋) 계산 결과가 float64가 됨. OpenCV의 drawContours, boundingRect, minAreaRect 등은 보통 정수(int32) 좌표를 기대하는데 float64가 들어가면 에러가 나고, 그 에러가 except로 잡히면서 render_main_view()가 호출되지 않아 화면이 갱신되지 않아 "멈춘 것처럼" 보임.
해결:
- 오프셋을 더한 뒤 반드시 정수로 바꿔서 넘김: .astype(np.int32). 예: contours = [(cnt + offset).astype(np.int32) for cnt in contours]

[원인 3 - 에러 나도 화면은 갱신]
무슨 일이었나: 위처럼 에러가 나면 except 블록에서 로그만 찍고 끝나서, current_display_frame이 안 바뀌고 render_main_view()도 안 불림 → 화면이 멈춘 것처럼 보임.
해결:
- except 블록에서도 full_frame 또는 raw_frame으로 current_display_frame을 세팅하고 render_main_view()를 한 번 호출해서, 에러가 나도 최소한 마지막 프레임은 보이게 함.
